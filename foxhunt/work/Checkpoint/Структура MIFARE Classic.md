# Спецификация карты — Mifare Classic 1K, FoxAss

---

## Общие параметры

- **Порядок байт:** Big Endian везде
- **Key A** = **Key B** = `HMAC-SHA256(master_secret, event_id)[0:6]` — одинаковый для всех 16 секторов (master_secret.key создается заранее)
- **Access bits** = `FF 07 80` — одинаковые для всех секторов;
- **GPB** = `69`
- **MAC** = `HMAC-SHA256(key, data)[0:4]` — пишется при каждой записи блока

---

## Сектор 0 — Идентификация участника

### Блок 0 — Manufacturer Block (read-only, заводской)

| Смещение | Размер | Поле | Описание |
|----------|--------|------|----------|
| 0 | 4б | `UID` | Уникальный идентификатор карты |
| 4 | 1б | `BCC` | Контрольный байт UID |
| 5 | 1б | `SAK` | Select Acknowledge |
| 6 | 2б | `ATQA` | Answer to Request |
| 8 | 8б | — | Данные производителя |

Не перезаписывается. UID используется во всех MAC как соль.

### Блок 1 — Данные участника

| Смещение | Размер | Поле             | Описание                                                                                                                  |
| -------- | ------ | ---------------- | ------------------------------------------------------------------------------------------------------------------------- |
| 0        | 2б     | `event_id`       | ID соревнования                                                                                                           |
| 2        | 2б     | `participant_id` | Номер участника                                                                                                           |
| 4        | 1б     | `group_id`       | Группа/класс (M21, W14 и т.д.)                                                                                            |
| 5        | 1б     | `format`         | Версия формата = `0x01`                                                                                                   |
| 6        | 1б     | `flags`          | Статусные флаги участника (см. ниже)                                                                                      |
| 7        | 1б     | резерв           | = `0x00`                                                                                                                  |
| 8        | 4б     | резерв           | = `0x00000000`                                                                                                            |
| 12       | 4б     | `MAC`            | `HMAC-SHA256(master_secret, event_id‖participant_id‖group_id‖format‖flags‖резерв‖резерв‖name‖UID)[0:4] - name из блока 2` |

#### flags\[1б\] — побитово

| Бит | Имя       | Смысл                                  |
| --- | --------- | -------------------------------------- |
| 0   | `START`   | 1 = старт взят (КП 0 записан)          |
| 1   | `FIN`     | 1 = финиш взят                         |
| 2   | `CARD_OK` | 1 = карта инициализирована корректно   |
| 3   | `DISQ`    | 1 = участник дисквалифицирован         |
| 4   | `EXTRA`   | 1 = на трассе есть КП с доп. заданиями |
| 5   | `DNS`     | 1 = не стартовал (Did Not Start)       |
| 6   | `DNF`     | 1 = не финишировал (Did Not Finish)    |
| 7   | резерв    | = 0                                    |

**Ограничения:** `DNS` и `START` взаимоисключающие. `DNF` и `FIN` взаимоисключающие.

## Секторы 1–14 — Отметки КП

Каждый сектор содержит **3 блока данных** (блоки 0, 1, 2) и **1 трейлер** (блок 3). Каждый блок данных — один слот КП.

### Адресация слотов

Слот с порядковым номером `(S−1)×3 + B` — физическая позиция в пуле (индекс 0–41), где S ∈ {1..14} — номер сектора, B ∈ {0, 1, 2} — номер блока внутри сектора.

`cp_num` — логический идентификатор КП, независимый от позиции слота. Один и тот же `cp_num` может встречаться дважды (дубль, `flags=0x01`); позиция слота всегда уникальна.

**Итого 42 слота** с индексами 0–41:

| `cp_num`     | Смысл         |
| ------------ | ------------- |
| `0x00`       | Старт         |
| `0x01..0xFD` | Обычные КП    |
| `0xFE`       | Финиш         |
| `0xFF`       | Слот свободен |

Слоты заполняются последовательно начиная с сектора 1, блока 0. Финишная станция записывает отметку `cp_num=0xFE` в очередной свободный слот пула **как рядовое КП** — никакого специального поведения не требуется.

---

### Блоки 0–2 каждого сектора — Отметка КП

| Смещение | Размер | Поле        | Описание                                                            |
| -------- | ------ | ----------- | ------------------------------------------------------------------- |
| 0        | 1б     | `cp_num`    | Номер КП. `0x00` = старт, `0xFE` = финиш, `0xFF` = слот пустой      |
| 1        | 8б     | `ts_arrive` | Unix timestamp прибытия, uint64 BE                                  |
| 9        | 1б     | `flags`     | Статус отметки (см. ниже)                                           |
| 10       | 2б     | `reserve`   | Данные станции (доп. задание). `0x0000` при 1-м касании             |
| 12       | 4б     | `MAC`       | `HMAC-SHA256(station_key, cp_num‖ts_arrive‖flags‖reserve‖UID)[0:4]` |

`station_key` — ключ конкретной станции, производный от `master_secret` и `cp_num`: `HMAC-SHA256(master_secret, cp_num)[0:16]`. Финишная станция использует `cp_num=0xFE` для деривации своего ключа.

#### flags\[1б\] — значения

| Значение | Смысл |
|----------|-------|
| `0x00` | Норма — отметка завершена |
| `0x01` | Дубль — повторное касание на уже закрытом КП |
| `0x02` | Ошибка — сбой при записи / некорректные данные |
| `0x03` | Ожидание — 1-е касание принято, доп. задание не выполнено |

#### Логика двухкасательного КП

**1-е касание:**
- `cp_num` = номер КП
- `ts_arrive` = текущий timestamp
- `flags = 0x03`
- `reserve = 0x0000`
- MAC вычисляется и записывается

**2-е касание** (задание выполнено):
- `cp_num` и `ts_arrive` не меняются (берутся из памяти КП, а не с карты)
- `flags = 0x00`
- `reserve` = данные станции
- MAC пересчитывается и перезаписывается

Финишный ридер верифицирует MAC при любом значении флага. Флаг `0x03` на финише означает, что участник не завершил доп. задание.

## Сектор 15 — Финиш и служебное

Сектор 15 заполняется **судейским ПО постфактум** — после того как участник сдал карту на финише. Финишная станция в сектор 15 **не пишет**.

### Блок 0 — Финиш

| Смещение | Размер | Поле          | Описание                                                                                                    |
| -------- | ------ | ------------- | ----------------------------------------------------------------------------------------------------------- |
| 0        | 8б     | `finish_time` | Unix timestamp финиша, uint64 BE                                                                            |
| 8        | 1б     | `status`      | `0x01` = финиш ок / `0x02` = снят / `0x03` = лимит                                                          |
| 9        | 1б     | `cp_count`    | Количество заполненных слотов в пуле (секторы 1–14), включая слот `0xFE`, по данным судейского сканирования |
| 10       | 2б     | резерв        | = `0x0000`                                                                                                  |
| 12       | 4б     | `MAC`         | `HMAC-SHA256(master_key, finish_time‖status‖cp_count‖UID)[0:4]`                                             |

### Блок 1 — Целостность

| Смещение | Размер | Поле | Описание |
|----------|--------|------|----------|
| 0 | 4б | `CRC32` | CRC32 от всех 42 блоков отметок КП (секторы 1–14, блоки 0–2) в порядке S1B0, S1B1, S1B2, S2B0, …, S14B2; каждый блок = 16 байт; итого 672 байта |
| 4 | 4б | `MAC` | `HMAC-SHA256(master_key, CRC32‖UID)[0:4]` |
| 8 | 8б | резерв | = `0x00…` |

### Блок 2 — Резерв

| Смещение | Размер | Поле |
|----------|--------|------|
| 0 | 16б | резерв = `0x00…` |

### Блок 3 — Sector Trailer

Аналогично сектору 0.
